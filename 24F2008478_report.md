## 1. Introduction
*   **What is the problem?** 
    The task is a multi-class thoracic pathology classification problem. Given a chest X-ray image, the model must identify which of the 20 specific pathological conditions (or "No Finding") is present.
*   **Why is it important?** 
    Automated chest X-ray screening assists radiologists in prioritizing urgent cases and reduces diagnostic errors, especially in high-volume clinical settings.
*   **What is the dataset?** 
    The dataset consists of chest X-ray images with a corresponding `train.csv` containing image IDs and binary labels for 20 classes. It includes metadata for training and a hidden test set for evaluation.
*   **What is your objective?** 
    To develop a deep learning pipeline that maximizes classification accuracy while specifically minimizing False Negatives, as penalized by the competition's asymmetric scoring rules.

## 2. Methodology
*   **Models Used:** 
    I experimented with **ResNet50**, **EfficientNet-B0**, and **DenseNet121**. 
*   **Why you chose them:** 
    **DenseNet121** was selected as the final architecture. Unlike ResNet, DenseNet’s dense connectivity pattern promotes feature reuse and is particularly effective at capturing the subtle, low-contrast patterns typical of medical imaging (like nodules or infiltrates).
*   **Preprocessing Steps:**
    *   **CLAHE (Contrast Limited Adaptive Histogram Equalization):** Used to enhance the contrast of the X-rays, making lung structures more visible.
    *   **Resizing:** All images were resized to 224x224 to match the ImageNet pre-trained input requirements.
    *   **Normalization:** Applied standard ImageNet mean and standard deviation for transfer learning stability.
*   **Techniques:**
    *   **Data Augmentation:** Implemented Random Horizontal Flips to increase model robustness.
    *   **Combined Asymmetric Loss:** Created a custom loss function merging **Weighted BCE** (penalizing False Negatives by 5x) and **Focal Loss** to handle class imbalance and hard-to-classify samples.
    *   **Transfer Learning:** Initialized weights from `IMAGENET1K_V1`.

## 3. Experiments
*   **Experiments Performed:** 
    Compared model performance across three different architectures (ResNet, EfficientNet, DenseNet) and tested different loss functions (Standard BCE vs. Custom Asymmetric Loss).
*   **Variations Tried:**
    *   **Learning Rate:** Tested 1e-3 (too aggressive, caused oscillations) vs. 1e-4 (stable convergence).
    *   **Batch Size:** Set to 32 to optimize memory usage on Kaggle's P100 GPU.
*   **Validation Strategy:** 
    Used an **85/15 Train-Validation split**. This ensures enough data for training while maintaining a representative set to monitor for overfitting.

## 5. Conclusion
*   **Summary:** 
    DenseNet121 combined with CLAHE preprocessing and a custom Weighted BCE loss provided the most reliable results. The model effectively handles the asymmetric penalty requirements of the competition.
*   **Key Learnings:** 
    Medical images require specific preprocessing (like CLAHE) compared to natural images. Furthermore, when a competition penalizes False Negatives heavily, a custom weighted loss function is more effective than standard Cross-Entropy.
